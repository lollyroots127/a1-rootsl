Web Dev Concepts

This file is named index.html, and it is the designated homepage of the rootsl/a1-rootsl directory located on the OSU Engineering web server. When a web browser sends an HTTP request for the directory, the Apache web server, the software used by OSU Engineering, maps the request to this file. The majority of web servers are configured to search for an index.html file whenever they receive a request for /, the root directory. On some web servers, other file names are accepted as homepages. For example, Microsoft web servers will also look for default.html files.

The Inspection tools in web browsers such as Firefox and Google Chrome allow users to view requests and responses between the web client and different servers. The network information can differ based on which protocol is used. When viewing the local file version of this webpage, the network information will show three resources: index.html, main.css, and main.js. The general network information for the index.html file shows the resource URL, request method, status code, and referrer policy. It shows limited response and request headers as well. When the same file is accessed through a web server, the network information is quite different. Three additional resources are shown: two stylesheets and favicon.ico. The general tab includes a remote IP address of where the server is located. The request and response headers are more detailed, showing information on the server, the user-agent, and time and date signatures.
